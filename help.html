üìñ Getting Started

What is ORA Automation?

ORA Automation is a fulfillment system that automatically processes orders from XML files, uploads them to ShipStation, tracks shipments, and manages inventory. The system runs 24/7 with minimal manual intervention.

How It Works

  1. Order Import: XML files are automatically imported from Google Drive every 5 minutes
  2. ShipStation Upload: Orders are uploaded to ShipStation with correct lot numbers
  3. Status Tracking: System monitors order status (pending ‚Üí awaiting shipment ‚Üí shipped)
  4. Inventory Update: When orders ship, inventory is automatically updated

Your Main Tasks

  • Monitor Dashboard: Check that orders are processing smoothly
  • Handle Alerts: Review shipping violations and inventory warnings
  • Generate Reports:
    • EOW (End of Week) - Weekly inventory report for manufacturer & customer
    • EOM (End of Month) - Monthly chargeback report for customer

‚ùì Frequently Asked Questions

How often do orders import automatically?

Every 5 minutes. The system polls Google Drive for new XML files continuously.

What if an order doesn't appear?

Go to Orders Inbox and click "Manual Import". Make sure the XML file is in the correct Google Drive folder.

When should I click EOD vs EOW vs EOM?

  • EOD (End of Day): Syncs shipped items - run daily or when needed
  • EOW (End of Week): Generates weekly inventory report - run every Friday
  • EOM (End of Month): Generates chargeback report - run at month-end

What's the difference between "ShipStation Units" and "Local DB Units"?

Local DB Units: Orders in ORA database ready to upload.
ShipStation Units: Orders already uploaded to ShipStation awaiting shipment.

Why is there a shipping violation?

The system detected an order shipped with incorrect carrier/service (e.g., Hawaiian orders must use FedEx 2Day). Review and resolve if needed.

Can I manually add an order?

Yes! Go to Orders Inbox and click "Manual Import". Enter order details and it will upload to ShipStation.

What happens if inventory runs out?

The system shows red warning badges (e.g., "11 days left"). Coordinate with manufacturer to restock before it hits zero.

üîß Troubleshooting

‚ùå Order shows "FAILED" status

Possible causes:

  • Missing or invalid SKU-Lot mapping
  • ShipStation API error
  • Duplicate order already exists

Solution:

  1. Check error message in Orders Inbox
  2. Verify SKU-Lot mapping exists (go to SKU Lot Management)
  3. Try "Sync Manual" to retry upload

‚è∏Ô∏è Orders stuck at "pending"

Possible causes:

  • Upload workflow paused
  • ShipStation connection issue

Solution:

  1. Check Workflow Controls - ensure "ShipStation Upload" is ON
  2. Wait 5 minutes (auto-upload cycle)
  3. Use "Sync Manual" to force upload

üìâ KPI numbers look wrong

Solution:

  1. Click the "Refresh" button in dashboard header
  2. Wait for auto-refresh (happens every 30 seconds)
  3. Check "Last updated" timestamp under each KPI

üî¥ System Status shows "Warning"

Solution:

  1. Click on the System Status card to see details
  2. Check if workflows are running (Workflow Controls page)
  3. Review recent error logs if available
