### Step 4: Create Your First Action Plan (70-80 minutes)

**Choose a feature:** Medium complexity (3-5 hours implementation)

**Follow the 8-step workflow:**

1. **Requirements** (10 mins) ‚Üí Define Section 1-2
2. **Gap Analysis** (15 mins) ‚Üí Analyze 5 context areas
3. **Gap Closure** (5 mins) ‚Üí Feed decisions back to Sec 1-2
4. **Optimization** (10 mins) ‚Üí Document trade-offs
5. **Risk Assessment** (15 mins) ‚Üí Assess 6 categories
6. **Deployment Planning** (10 mins) ‚Üí Create checklist
7. **Executive Summary** (5 mins) ‚Üí Write for stakeholders
8. **Final Review** (5 mins) ‚Üí Run readiness checklist

**Use:** `ACTION-PLANNING-QUICK-REFERENCE.md` as your guide

---

### Step 5: Execute with Checklist (Varies by feature)

**Generate execution checklist:**
1. Copy `EXECUTION-CHECKLIST-TEMPLATE.md`
2. Rename to `[FEATURE-NAME]-EXECUTION-CHECKLIST.md`
3. Fill in time estimates from action plan Section 6
4. Follow phase-by-phase during implementation

**8 Phases:**
1. Pre-Implementation Review (5-10 mins)
2. Setup & Foundation
3. Core Implementation
4. Integration
5. Testing & QA
6. Risk Mitigation Verification
7. Documentation
8. Deployment & Verification

---

### Step 6: Measure & Improve (10-15 minutes)

**After implementation:**
1. Track actual vs estimated time
2. Calculate quality metrics (from procedure doc)
3. Note what took longer than expected
4. Update templates based on learnings
5. Document improvements for next feature

## üö® Common Pitfalls (From Real Experience)

### ‚ùå Pitfall #1: Skipping Gap Closure Feedback Loop
**Fix:** Always complete Section 3.7 showing where gaps updated Sections 1-2

### ‚ùå Pitfall #2: Vague Trade-off Reasoning
**Fix:** Add "Reasoning" explaining each trade-off decision

### ‚ùå Pitfall #3: Incomplete Risk Mitigation
**Fix:** Every risk must have mitigation strategy (use Section 5 checklist)

### ‚ùå Pitfall #4: Placeholder Text in Final Plan
**Fix:** Run final readiness review - no TODO/TBD allowed

### ‚ùå Pitfall #5: Time Estimates Only as Totals
**Fix:** Break down by phase: "15+7+5+3 mins" not "30 mins"

### ‚ùå Pitfall #6: Technical Jargon in Executive Summary
**Fix:** Write for a smart 12-year-old (clear, simple language)
